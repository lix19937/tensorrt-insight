## 敏感度分析（Layer-wise Sensitivity Analysis）逻辑。 

### 1 单层逐层扰动法        
* 找到所有 quant layer   
* 每次仅使能一层quant layer进行指标eval **基于实际任务指标(mAP/Acc)** ，记录到dict中 {"layer_name":eval_value}
* 如果是使用 pytorch-quantization calib，则是在pytorch 下寻找敏感层
* 如果是使用 TensorRT calib，则是在 fp32 下，使用 TensorRT 或 onnxruntime 下寻找

其核心思想是：“一次只量化一个算子，其余算子保持 FP32 精度”，通过观察不同算子单独量化时对模型指标（如 mAP）的影响，来评估哪些层对量化最敏感。    
控制变量法：通过循环开关，该脚本成功地将“模型整体量化误差”解耦为“各个单层对误差的贡献值”。这在进行模型压缩调试时非常有效。

灵活性：它允许开发者根据 evaler.eval 返回的结果，直接可视化出“哪一层导致了精度骤降”。

### 待改善点

未考虑“量化级联效应”：        
该方法只能测量单层敏感度，无法捕捉“多层同时量化”时的误差累积。有些模型单层量化没事，但连续两层量化会导致精度崩溃。    

效率问题：     
对于大型网络（如 YOLO 或 ResNet），该代码需要循环运行 $N$ 次（$N$ 为可量化层数）。如果 evaler.eval 非常耗时（例如在全验证集上跑 mAP），整个分析过程会极其缓慢。
建议：在分析阶段，建议只在一小部分验证数据（Subset）上进行评估，而不是在全量测试集上。

算子匹配范围：     
当前的 if 判断只涵盖了卷积和池化。
如果模型包含 Attention 层（如 Transformer）或特殊的激活函数层，该脚本可能会漏掉对这些层的敏感度分析。
根据生成的 quant_sensitivity 列表绘制一个柱状图（横轴为层名，纵轴为 mAP 跌落幅度）。如果有某一层在量化后 mAP 出现断崖式下跌，那该层就是您的“敏感层”，可以考虑将其强制锁定为 FP32。

在实际操作中，除了简单地把这些层设回 FP32，您还可以尝试以下几种策略来优化精度：      

混合精度（Mixed Precision）：将最敏感的 5%-10% 的层保留为 FP32，其余层量化为 INT8。这通常能在保持模型大小和速度优势的同时，最大程度恢复精度。    

校准数据优化（Calibration Optimization）：如果某一层的敏感度很高，有时是因为该层在校准阶段（Calibration）未被充分覆盖。尝试增加校准数据集的多样性。     

权重重构（Weight Re-quantization）：对于特定的敏感层，可以使用更精细的量化算法（如 GPTQ 或 AWQ 等）进行额外优化。

单层逐层扰动法”分析结果不稳定，或者执行时间太长：
先尝试“特征图分布距离（MSE）”：这是最容易落地的。您只需要在您的代码循环中加入一个钩子（Hook），截取某一层量化前后的 output，对比它们的 MSE。如果某层的 MSE 显著高于其他层，那它通常就是敏感层。

### 2 Hessian 矩阵近似法 (Fisher Information / Hessian Trace)    
这是目前学术界和工业界最推崇的数学手段。其核心逻辑是：**模型的损失函数对某一层权重的“二阶导数”（Hessian 矩阵）** 决定了该层对量化的敏感度。    
原理：如果损失函数在某个方向上的曲率（Curvature）很大，说明在该方向上进行微小的扰动（量化噪声）会引发巨大的损失变化。    

优势：不需要像您那样对每一层进行实测（无需反复 eval），通过计算一段数据上的 Hessian 迹（Trace）或 Fisher 信息矩阵即可得出每层的敏感度，速度极快。  

代表算法：GPTQ、AWQ（虽然它们主要用于权重压缩，但其核心都是基于对 Hessian 的近似来量化重要性排序）。     

### 3 特征图分布对比法 (KL 散度 / MSE 距离)如果您不想依赖任务指标（如 mAP），可以完全在中间层进行分析          
原理：量化前后的特征图（Feature Map）在分布上越接近，敏感度越低。    

具体实现：      
在推理时，记录第 $L$ 层在 FP32 模式下的激活分布（Activation Distribution）。  
量化该层后，记录相同的激活分布。  
计算两者之间的 KL 散度 或 均方误差（MSE）。    

优势：完全不需要标注数据集，只需要一部分校准数据即可计算，适合无监督或缺乏标签的场景。    

### 4 量化误差累积分析 (Error Propagation Analysis)      
“单层独立量化”，但这忽略了误差的级联反应    
原理：计算第 $i$ 层的量化噪声如何随网络传播并最终影响到输出。   

具体实现：   
在每层输出端显式加入高斯噪声（模拟量化噪声），并在输出端监测误差。  
通过反向传播计算误差对于各层量化参数的梯度。   

优势：能够发现某些层虽然单层表现不敏感，但因为处于网络关键路径（如 Shortcut 连接或注意力计算分支）上，从而放大了下游误差。
